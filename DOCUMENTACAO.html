Cooperativa - Controle de Ativos e Inventário

1. Visão Geral do Sistema

O CoopAssets é uma aplicação web voltada para o controle e rastreamento de equipamentos e ativos de uma cooperativa. O sistema permite o cadastro de funcionários, o registro de objetos (notebooks, veículos, ferramentas) e o controle exato de quem está com qual equipamento em determinado momento temporal.

Figura 1: Dashboard interativo com Single Page Application (SPA)

Arquitetura e Tecnologia

O sistema foi migrado de uma estrutura local estática para uma arquitetura Cliente-Servidor robusta, utilizando as seguintes tecnologias:

Camada Tecnologia / Detalhes
Backend (Servidor) Python 3 e Django Framework. Responsável por expor as APIs REST, lidar com segurança e processar planilhas Excel.
Banco de Dados SQLite3 conectado nativamente ao ORM do Django. Garante a integridade e proteção estruturada dos dados.
Frontend (Cliente) HTML5, CSS3 Customizado (Glassmorphism) e Vanilla JavaScript (Fetch API) operando em modelo SPA (página única).

A Revolução do Fetch()

O que mudou? Antes os dados ficavam limitados ao seu navegador (LocalStorage). Agora, o JavaScript opera como um "consumidor remoto". Ao clicar em um botão, o Javascript congela, faz um await fetch('/api/assets') mandando um pacote para o Python. O Python trabalha com o Banco de dados e devolve tudo limpo em formato JSON instantaneamente. Isso permite múltiplas máquinas usarem o sistema simultaneamente.

2. Banco de Dados e APIs

Modelos de Dados (Models)

A estrutura de dados relacional (SQL) foi desenhada com três tabelas principais dentro do arquivo models.py.

  • Asset (Ativos): Guarda o nome, a categoria e gera um ID único criptográfico aleatório (ex: asset-1628...) toda vez que é criado. Esse ID alimenta a geração de QR Codes.
  • Employee (Funcionários): Guarda o nome e o setor/cargo.
  • Assignment (Empréstimos): A tabela pivô mais importante. É ela que possui uma Chave Estrangeira (ForeignKey) para o Funcionário e outra para o Ativo. Ela possui um status que indica se o empréstimo está active (Pessoa usando) ou se já foi returned (Devolvido para a gaveta).

Estrutura Endpoint API Base

As rotas configuradas no views.py trabalham de forma fluida dependendo do verbo HTTP da requisição:

GET /api/assets/ -> Retorna a lista de todos os ativos em JSON POST /api/assets/ -> Salva um novo ativo no banco recebendo JSON DELETE /api/assets/<id> -> Exclui um ativo (Bloqueia se estiver 'Em Uso') POST /api/assignments/ -> Cria um link 'active' entre Ativo e Funcionário POST /api/assignments/<id>/return/ -> Muda o status do empréstimo para 'returned'

3. Recursos Avançados

3.1. Importação e Exportação (Excel)

O sistema utiliza a biblioteca Openpyxl sob demanda para lidar com grandes massas de dados com o pacote Office.

Exportar (.xlsx)

Ao requisitar /api/export-excel/, o Python cria um Workbook de Excel "em memória" no servidor, laça e escreve todos os ativos com cabeçalho (Nome, Categoria, Status, etc). Em seguida ele empacota no formato binário application/vnd.openxmlformats e envia forçando o download no seu Microsoft Excel.

Importar (.xlsx)

O front-end possui um input invisível mascarado pelo botão. Ao enviar a planilha por POST formulário para a API /api/import-excel/, o Python abre a planilha inteira, ignora as linhas vazias e cabeçalhos, e extrai todos os Nomes da Coluna B e Categorias da Coluna C, varrendo o loop para salvar centenas de registros em dois segundos usando a classe Model.

3.2. Gerador e Leitura de QR Code Automático

O código QR não gasta memória da sua máquina gerando pesadas imagens em Base64. Utilizamos uma técnica Stateless:

const qrUrl = `https://api.qrserver.com/v1/create-qr-code/?size=200x200&data=${assetId}`;

Ele entrega apenas o ID único para a placa de API Online que nos devolve instantaneamente um PNG enquadrado com a margem correta. Ao simular na aba "Leitura" inserindo o ID na busca, o Python varre instantaneamente a tabela Assignment procurando o "Ativo=X" com o Status "Active" e retorna a Ficha Cadastral da pessoa portadora e a data exata em sua mão.
